# Terms of Service Implementation

## Обзор изменений

В систему экзаменов добавлено **Пользовательское соглашение (Terms of Service)** перед процессом регистрации.

### Что было добавлено:

1. **Пользовательское соглашение на 3 языках**
   - Английский (English)
   - Русский (Russian)  
   - Иврит (Hebrew)

2. **Профессиональный юридический дисклеймер**
   - Отказ от ответственности за потерю данных
   - Отказ от гарантий (freeware)
   - Ограничение ответственности (гражданская, уголовная, судебная)
   - Защита YaM SOFT от любых исков

3. **UI компоненты**
   - Чекбокс для согласия с условиями на странице регистрации
   - Модальное окно для чтения полного текста соглашения
   - Кнопка "Принять и продолжить" в модальном окне
   - Автоматическая установка чекбокса при нажатии "Принять"

4. **База данных**
   - Новое поле `terms_accepted_at` в таблице `users`
   - Сохранение даты и времени принятия условий
   - Скрипт миграции для обновления существующей БД

### Основные разделы соглашения:

1. **Принятие условий** - обязательное согласие для использования сервиса
2. **Описание сервиса** - платформа предоставляется "AS IS"
3. **Отказ от ответственности** - нет ответственности за:
   - Потерю данных, паролей, информации
   - Сбои системы, простои
   - Точность результатов экзаменов
   - Любые прямые или косвенные убытки
4. **Отсутствие гарантий** - продукт freeware без каких-либо гарантий
5. **Данные и конфиденциальность** - данные могут быть удалены в любой момент
6. **Обязанности пользователя** - ответственность за пароли и контент
7. **Прекращение обслуживания** - право удалить аккаунт без предупреждения
8. **Ограничение ответственности** - никакой гражданской/уголовной/судебной ответственности
9. **Изменения условий** - право изменять без уведомления
10. **Применимое право** - местная юрисдикция

### Технические детали:

**Файлы, которые были изменены:**
- `templates/signup.html` - добавлен чекбокс и модальное окно
- `app.py` - добавлена проверка согласия с условиями
- `services/auth_service.py` - добавлено поле terms_accepted_at
- `data/translations/en.json` - английские переводы
- `data/translations/ru.json` - русские переводы  
- `data/translations/he.json` - ивритские переводы

**Новые файлы:**
- `update_db_schema.py` - скрипт для миграции базы данных
- `TERMS_OF_SERVICE_README.md` - этот файл

### Как это работает:

1. Пользователь открывает страницу регистрации
2. Видит чекбокс "Я прочитал и принимаю Условия использования"
3. Может кликнуть на ссылку "Читать Условия использования"
4. Открывается модальное окно с полным текстом на выбранном языке
5. Может нажать "Принять и продолжить" - чекбокс автоматически установится
6. Без установленного чекбокса регистрация невозможна
7. При регистрации в БД сохраняется timestamp принятия условий

### Миграция существующей БД:

Если у вас уже есть пользователи в базе данных, запустите:

```bash
python update_db_schema.py
```

Это добавит новое поле `terms_accepted_at` в таблицу users.

### Поддержка языков:

Система автоматически показывает текст соглашения на выбранном языке:
